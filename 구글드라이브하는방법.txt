OAuth 클라이언트 ID 발급 (신분증 만들기)
Google Cloud Console에 접속하여 프로젝트를 만듭니다.

내 앱의 패키지 이름과 **디지털 서명(SHA-1)**을 등록하여 전용 Client ID를 발급받습니다.

이 ID를 googleDrive.js 파일의 상단 CLIENT_ID 칸에 넣어야 합니다.

2. OAuth 동의 화면 구성 (가게 간판 달기)
사용자가 로그인할 때 보게 될 앱 로고, 이름, 개발자 이메일을 등록합니다.

여기서 **범위(Scopes)**를 설정해야 하는데, drive.file 권한을 선택해야 합니다. (이 앱이 만든 파일에만 접근한다는 뜻입니다.)

3. 구글 심사 (경비실 승인)
모든 설정을 마친 후 [앱 게시] 버튼을 눌러 심사를 요청합니다.

구글 직원이 "이 앱이 사용자의 드라이브 권한을 왜 쓰는지" 확인합니다.

심사 기간: 보통 2~3일에서 길게는 1~2주 정도 걸립니다. 승인이 완료되면 "위험한 앱" 경고가 사라집니다.

💡 "자동 백업"은 어떻게 구현하나요?
심사가 통과된 후, 사장님이 원하시는 **"조용한 자동 백업"**을 만들려면 코드를 아주 살짝만 추가하면 됩니다.

방법: App.js에서 앱이 켜질 때나 꺼질 때, 혹은 하루에 한 번씩 backupToDrive(entries) 함수를 자동으로 실행하게 만들면 됩니다.

사용자 경험: 사용자는 백업 버튼을 누를 필요도 없이, 앱을 쓰다 보면 자기 구글 드라이브의 **'AppData'**라는 숨겨진 폴더에 데이터가 안전하게 보관됩니다.

⚠️ 주의사항
테스트 모드: 심사 전에는 사장님 계정으로만 테스트가 가능합니다.

개인정보 처리방침: 구글 심사 시 개인정보 처리방침 URL을 요구합니다. 배포용 홈페이지나 블로그 등에 내용을 올려두고 링크를 제출해야 합니다.

이제 구글 클라우드 콘솔 설정을 시작할 준비가 되셨나요? 필요하시다면 설정 단계별로 눌러야 할 메뉴들을 하나씩 안내해 드릴 수 있습니다.
